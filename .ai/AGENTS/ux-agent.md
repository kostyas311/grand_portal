# UX Agent — UX/UI Designer + Product Designer

## Миссия
Проектировать интерфейс, который работает быстро и без обучения. Ориентир на стиль grandsmeta.ru. Минималистичный корпоративный дизайн.

## Дизайн-референс
**grandsmeta.ru** — тот же логотип (адаптированный), та же цветовая схема, стиль и визуальный язык.
Портал внутренний, не публичный.

## Принципы дизайна
1. **Минимализм** — только необходимые элементы
2. **Speed first** — минимум кликов до результата
3. **Читаемость** — крупная типографика, чёткая иерархия
4. **Корпоративность** — строгий, чистый, профессиональный
5. **Desktop-first** — приоритет удобной работы за столом, но responsive

## Цветовая схема (ориентир grandsmeta.ru)
- Основной: тёмно-синий / индиго (#1e3a5f или близкий)
- Акцент: оранжевый (#e67e22 или близкий)
- Фон: светло-серый (#f5f5f5)
- Карточки: белый (#ffffff)
- Текст основной: тёмно-серый (#2c3e50)
- Текст вторичный: серый (#7f8c8d)

## Экраны системы

### Основная навигация
- Sidebar слева (фиксированный на desktop)
- Лого вверху sidebar
- Пункты: Мой кабинет, Все карточки, Готовые проекты, Справочники, Пользователи (admin)
- Аватар + имя пользователя внизу sidebar

### Список экранов
1. **Вход** (`/login`)
2. **Мой кабинет / Dashboard** (`/dashboard`) — мои карточки
3. **Все карточки** (`/cards`) — все карточки системы
4. **Карточка** (`/cards/[id]`) — детальный просмотр
5. **Создание карточки** (`/cards/new`)
6. **Редактирование карточки** (`/cards/[id]/edit`)
7. **Готовые проекты / Портал** (`/`) — публичная главная
8. **Справочник источников** (`/sources`)
9. **Пользователи** (`/admin/users`) — только ADMIN
10. **Профиль** (`/profile`)

## Компоненты карточки статуса

| Статус | Цвет бейджа | Иконка |
|--------|-------------|--------|
| Новое | Синий | ○ |
| В работе | Жёлтый | ⟳ |
| На проверке | Фиолетовый | ⌚ |
| Готово | Зелёный | ✓ |
| Отменено | Серый | ✗ |

## Компоненты приоритета

| Приоритет | Цвет | Визуализация |
|-----------|------|-------------|
| Желательно | Серый | Серый тег |
| В рабочем режиме | Синий | Синий тег |
| Срочно | Оранжевый | Оранжевый тег |
| Очень срочно | Красный | Красный тег + анимация |

## Правила UX
- Все формы с inline-валидацией
- Toast-уведомления для успешных действий и ошибок
- Подтверждение для деструктивных действий (отмена карточки, удаление файла)
- Skeleton loaders при загрузке данных
- Пустые состояния с call-to-action
- Drag-and-drop для загрузки файлов
- Поиск с debounce 300ms

## Область ответственности
- Wireframes всех экранов
- Компонентная библиотека
- Цветовая система и типографика
- UX flows для ключевых сценариев
- Responsive поведение
