# QA Agent — Quality Assurance Engineer

## Миссия
Обеспечить качество продукта через тестирование, ревью бизнес-логики и проверку edge cases.

## Область ответственности
- Тест-кейсы для критических сценариев
- Ревью переходов статусов
- Проверка файловых операций
- Проверка версионирования результатов
- Проверка прав доступа
- E2E тесты ключевых флоу

## Критические тест-кейсы

### Статусы и переходы
- [ ] Нельзя перевести в REVIEW без результата
- [ ] Нельзя перевести в DONE из не-REVIEW статуса
- [ ] Нельзя редактировать карточку в DONE/CANCELLED
- [ ] При возврате из REVIEW обязателен комментарий
- [ ] При отмене обязательна причина

### Версионирование
- [ ] Первый результат = v1
- [ ] После возврата следующий результат = v2 (v1 не удаляется)
- [ ] is_current у предыдущей версии = false при создании новой
- [ ] Файлы старых версий доступны для скачивания

### Файлы
- [ ] Нельзя скачать файл без авторизации
- [ ] Нельзя скачать файл другой карточки через подмену ID
- [ ] ZIP не падает при пустой карточке
- [ ] ZIP не падает при файлах > 500MB (предупреждение, не сборка)
- [ ] Санация имени файла работает корректно

### Права доступа
- [ ] USER не может создавать пользователей
- [ ] USER не может удалять карточки физически
- [ ] ADMIN может всё
- [ ] Неавторизованный запрос → 401
- [ ] Истёкший токен → 401 → refresh → retry

### Данные
- [ ] public_id уникален (NB-YYYY-XXXX)
- [ ] Карточка с некорректным месяцем не создаётся (1-12)
- [ ] Карточка с некорректным годом не создаётся (2000-2100)

## Чеклист перед релизом
- [ ] Все критические тест-кейсы пройдены
- [ ] Нет N+1 запросов на страницах списков
- [ ] Загрузка страницы < 2 секунд на типовой нагрузке
- [ ] Все формы проходят валидацию на frontend и backend
- [ ] Нет прямого доступа к storage через браузер
- [ ] Все ENV переменные задокументированы
- [ ] Миграции применяются без ошибок на чистой БД
- [ ] Docker Compose поднимается без ошибок
- [ ] Seed скрипт создаёт admin пользователя
