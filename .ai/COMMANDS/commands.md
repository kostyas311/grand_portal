# Команды Product Owner — NormBase Portal

Используй эти команды в разговоре с Claude Code для управления разработкой.

---

## /new-feature

**Описание:** Реализовать новую функцию
**Шаблон:**
```
/new-feature [описание функции на человеческом языке]
```
**Пример:**
```
/new-feature добавить экспорт списка карточек в Excel
```
**Ожидаемый результат:**
- PM Agent формализует требование
- Architect Agent принимает технические решения
- Frontend + Backend агенты реализуют
- QA Agent проверяет

---

## /improve-ux

**Описание:** Улучшить удобство использования конкретного экрана или потока
**Шаблон:**
```
/improve-ux [экран или поток]
```
**Пример:**
```
/improve-ux форма создания карточки — сделать быстрее
```
**Ожидаемый результат:**
- UX Agent анализирует текущий экран
- Предлагает и реализует улучшения
- Сохраняет принципы минимализма и grandsmeta стиль

---

## /redesign-screen

**Описание:** Полностью переработать экран
**Шаблон:**
```
/redesign-screen [название экрана] [требования к новому дизайну]
```
**Пример:**
```
/redesign-screen главная страница карточек — сделать более компактный список
```

---

## /add-field

**Описание:** Добавить новое поле к карточке или другой сущности
**Шаблон:**
```
/add-field [сущность] [название поля] [описание]
```
**Пример:**
```
/add-field карточка "регион" — выпадающий список регионов из справочника
```

---

## /change-workflow

**Описание:** Изменить бизнес-процесс (статусы, переходы, правила)
**Шаблон:**
```
/change-workflow [описание изменения]
```
**Пример:**
```
/change-workflow добавить промежуточный статус "согласование" между "в работе" и "проверка"
```
**Ожидаемый результат:**
- PM Agent анализирует влияние на бизнес-процесс
- Architect Agent обновляет схему и API
- Frontend Agent обновляет UI
- QA Agent обновляет тест-кейсы
