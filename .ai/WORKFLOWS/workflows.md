# AI-команда: Рабочие процессы (Workflows)

## WORKFLOW-01: new-feature

**Триггер:** `/new-feature` или запрос Product Owner на новую функцию

```
1. PM Agent
   → Анализирует требование
   → Формализует user story + acceptance criteria
   → Определяет affected сущности и экраны
   → Фиксирует допущения

2. Architect Agent
   → Оценивает влияние на схему БД (нужны ли миграции)
   → Определяет новые/изменённые API endpoints
   → Принимает архитектурные решения

3. Backend Agent
   → Реализует Prisma миграцию (если нужна)
   → Реализует бизнес-логику в сервисе
   → Реализует API endpoint
   → Добавляет аудит лог

4. Frontend Agent
   → Реализует UI компонент(ы)
   → Подключает к API через TanStack Query
   → Добавляет валидацию форм

5. QA Agent
   → Формулирует тест-кейсы
   → Проверяет edge cases
   → Проверяет права доступа

6. PM Agent
   → Обновляет документацию
   → Отмечает фичу как реализованную
```

## WORKFLOW-02: bugfix

**Триггер:** `/fix-bug` или сообщение об ошибке

```
1. QA Agent
   → Воспроизводит ошибку
   → Определяет root cause
   → Оценивает severity

2. Architect Agent (если нужно)
   → Оценивает системное влияние
   → Принимает решение об архитектурном fix vs workaround

3. Backend / Frontend Agent
   → Реализует исправление
   → Пишет тест на регрессию

4. QA Agent
   → Проверяет исправление
   → Проверяет отсутствие регрессий
```

---

## WORKFLOW-03: screen-redesign

**Триггер:** `/redesign-screen` или `/improve-ux`

```
1. UX Agent
   → Анализирует текущий экран
   → Определяет проблемы UX
   → Проектирует новый layout (описание + логика)
   → Проверяет соответствие grandsmeta стилю

2. Frontend Agent
   → Реализует новый дизайн
   → Сохраняет функциональность

3. UX Agent
   → Проверяет реализацию против принципов
```

---

## WORKFLOW-04: schema-change

**Триггер:** изменение структуры данных

```
1. Architect Agent
   → Проектирует изменение схемы
   → Оценивает влияние на существующие данные
   → Пишет Prisma миграцию

2. Backend Agent
   → Обновляет сервисы и DTO
   → Обновляет API

3. Frontend Agent
   → Обновляет типы и компоненты

4. QA Agent
   → Проверяет миграцию на тестовых данных
```

---

## WORKFLOW-05: release-prep

**Триггер:** `/prepare-release`

```
1. QA Agent
   → Прогоняет полный чеклист
   → Документирует найденные проблемы

2. Backend Agent
   → Финализирует миграции
   → Проверяет ENV документацию

3. Infra Agent
   → Проверяет Docker Compose
   → Обновляет инструкцию деплоя
   → Проверяет бэкап-стратегию

4. PM Agent
   → Обновляет CLAUDE.md статус
   → Формирует changelog
```
