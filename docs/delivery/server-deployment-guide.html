Инструкция по развёртыванию

Полный запуск всех сервисов проекта в двух вариантах: на Ubuntu Server и на Windows Server / обычном ПК с Windows. Для примеров в инструкции используй шаблонный адрес вида <ваш-субдомен>.grandsmeta.ru.

Сценарий
Развёртывание на сервере
ОС
Ubuntu Server и Windows Server / Windows
Домен
<ваш-субдомен>.grandsmeta.ru
Важно
В production нужен HTTPS

1. Что будет установлено

  • PostgreSQL — база данных.
  • NormBase API — backend на NestJS.
  • NormBase Web — frontend на Next.js.
  • Nginx — reverse proxy, который будет принимать запросы снаружи.
  • systemd — для автозапуска API и Web после перезагрузки сервера.
Очень важно: у проекта refresh-cookie выставляются как Secure. Это значит, что на боевом сервере нужно открывать портал по https://, а не по обычному http://.
Ниже можно переключаться между вкладками Linux и Windows. В каждой вкладке приведены именно свои команды, свои пути к файлам и свои примеры настройки.

19. Настрой Nginx

sudo nano /etc/nginx/sites-available/normbase
server {
    listen 80;
    server_name <ваш-субдомен>.grandsmeta.ru;

    client_max_body_size 512M;
    client_body_timeout 300s;
    proxy_read_timeout 300s;
    proxy_send_timeout 300s;

    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "strict-origin-when-cross-origin" always;

    location /api/ {
        rewrite ^/api/?(.*)$ /$1 break;
        proxy_pass http://127.0.0.1:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_buffering off;
        add_header Cache-Control "no-store, no-cache, must-revalidate" always;
        add_header Pragma "no-cache" always;
    }

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

Включи конфиг:

sudo ln -s /etc/nginx/sites-available/normbase /etc/nginx/sites-enabled/normbase
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl restart nginx

25. Отдельный сценарий для Windows Server или обычного ПК с Windows

Важно: развёртывание на Windows подходит для пилота, внутреннего стенда, демонстрации или небольшого внедрения. Для максимально стабильного production-сценария всё равно предпочтительнее Ubuntu Server. Но если у вас инфраструктурно удобнее Windows, портал можно запустить и так.

25.1. Что понадобится на Windows

  • ОС: Windows Server 2019/2022 или Windows 10/11 Pro.
  • Права: локальный администратор на машине.
  • Сеть: статический IP или корректная DNS-запись на сервер.
  • Домен: например <ваш-субдомен>.grandsmeta.ru.
  • HTTPS: обязательно для нормальной работы авторизации.

25.2. Создай рабочие папки

mkdir C:\NormBase
mkdir C:\NormBase\storage
mkdir C:\NormBase\logs
mkdir C:\NormBase\tools

Дальше считаем, что проект будет жить в папке C:\NormBase\app.

25.3. Установи необходимые программы

  1. Установи Git for Windows.
  2. Установи Node.js 20 LTS.
  3. Установи PostgreSQL 16 или совместимую версию.
  4. Скачай Nginx for Windows и распакуй, например, в C:\nginx.
  5. Скачай NSSM (Non-Sucking Service Manager), чтобы красиво зарегистрировать API, Web и Nginx как службы Windows.
Почему нужен NSSM: без него можно запускать портал вручную в нескольких окнах PowerShell, но после перезагрузки компьютера всё придётся поднимать руками. NSSM позволяет сделать это как нормальные службы Windows.

25.16. Настрой HTTPS на Windows

Для Windows самый практичный путь — использовать win-acme или корпоративный сертификат вашей компании.

  1. Скачай win-acme.
  2. Выпусти сертификат на <ваш-субдомен>.grandsmeta.ru.
  3. Подключи сертификат в Nginx.
  4. Сделай редирект с http:// на https://.
Если сертификат не настроен, логин и refresh-cookie на production-домене могут работать некорректно. На боевом контуре HTTPS обязателен.

27. Частые проблемы

Сайт не открывается

sudo nginx -t
sudo systemctl status nginx

Не работает логин

Почти всегда причина в том, что заходят по http://, а не по https://.

API не стартует

sudo journalctl -u normbase-api -f

Web не стартует

sudo journalctl -u normbase-web -f

Не загружаются файлы

Проверь права на /opt/normbase/storage и директиву client_max_body_size в Nginx.

Не работают email-уведомления

28. Что желательно сделать обязательно

  • настроить резервное копирование PostgreSQL;
  • настроить резервное копирование папки /opt/normbase/storage;
  • проверять свободное место на диске;
  • использовать отдельный служебный email для администратора;
  • после развёртывания пройти тестовый сценарий: карточка -> результат -> проверка -> уведомление.
  • после первого входа администратор должен в личном кабинете настроить email-рассылку для домена grandsmeta.ru и проверить её через кнопку “Тест соединения”.
Если все шаги выше выполнены, портал должен стабильно работать по адресу https://<ваш-субдомен>.grandsmeta.ru.
Настройка email-рассылки не делается на уровне файлов проекта. Это отдельный обязательный шаг уже внутри портала: администратор входит в систему, открывает свой кабинет, переходит в настройки email-уведомлений и заполняет параметры почтового сервера для домена grandsmeta.ru.
