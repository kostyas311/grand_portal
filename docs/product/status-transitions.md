# Матрица статусов и переходов

## Статусы карточки

| Код | Отображение | Цвет | Описание |
|-----|-------------|------|---------|
| `NEW` | Новое | Синий #3B82F6 | Карточка создана, ещё не взята в работу |
| `IN_PROGRESS` | В работе | Жёлтый #F59E0B | Исполнитель работает над задачей |
| `REVIEW` | На проверке | Фиолетовый #8B5CF6 | Результат загружен, ожидает проверяющего |
| `DONE` | Готово | Зелёный #10B981 | Проверка пройдена, карточка закрыта |
| `CANCELLED` | Отменено | Серый #6B7280 | Карточка отменена, в архиве |

## Граф переходов

```
[NEW] ──────────────────────────────→ [CANCELLED]
  │
  ▼
[IN_PROGRESS] ──────────────────────→ [CANCELLED]
  │
  ▼ (обязательно: загружен результат)
[REVIEW]
  │
  ├──→ [DONE]        ← проверяющий: "Проверено, принято"
  │
  └──→ [IN_PROGRESS] ← проверяющий: "Обнаружены ошибки" + обязательный комментарий
```

## Детальные правила переходов

### NEW → IN_PROGRESS
- **Кто может:** ADMIN, COORDINATOR, EXECUTOR (если назначен исполнителем)
- **Условия:** нет дополнительных ограничений
- **Что происходит:** фиксируется дата начала, создаётся запись в истории

### IN_PROGRESS → REVIEW
- **Кто может:** ADMIN, COORDINATOR, EXECUTOR (если назначен исполнителем)
- **Условия:**
  - Карточка имеет хотя бы один результат (файл или ссылка)
  - Назначен проверяющий
- **Что происходит:** создаётся запись в истории, текущий ответственный — проверяющий

### REVIEW → DONE
- **Кто может:** ADMIN, COORDINATOR, REVIEWER (если назначен проверяющим)
- **Условия:**
  - Существует актуальная версия результата (is_current = true)
- **Что происходит:** карточка закрывается (locked = true), дата завершения фиксируется, карточка появляется на главной странице портала

### REVIEW → IN_PROGRESS (возврат с замечаниями)
- **Кто может:** ADMIN, COORDINATOR, REVIEWER (если назначен проверяющим)
- **Условия:**
  - Обязателен комментарий с описанием замечаний
- **Что происходит:** создаётся комментарий проверки, история фиксирует action_type=RETURNED, карточка возвращается исполнителю. Следующий загруженный результат = новая версия.

### ANY → CANCELLED
- **Кто может:** ADMIN, COORDINATOR
- **Условия:**
  - Карточка не находится в статусе DONE (завершённую нельзя отменить)
  - Обязательна причина отмены
- **Что происходит:** карточка архивируется (is_archived = true), locked = true, дальнейшее редактирование невозможно

## Блокировка редактирования

Карточка становится **locked = true** при переходе в:
- `DONE` — полностью заблокирована
- `CANCELLED` — полностью заблокирована

Для заблокированных карточек:
- Запрещено изменять любые поля
- Запрещено добавлять/удалять файлы и материалы
- Запрещено добавлять результаты
- Разрешено только скачивание файлов и просмотр

## Приоритеты

| Код | Отображение | Визуализация |
|-----|-------------|-------------|
| `OPTIONAL` | Желательно | Серый тег |
| `NORMAL` | В рабочем режиме | Синий тег |
| `URGENT` | Срочно | Оранжевый тег |
| `CRITICAL` | Очень срочно | Красный тег + иконка |

## Срок исполнения — фильтры

| Фильтр | Условие |
|--------|---------|
| Просрочено | `due_date < today AND status NOT IN (DONE, CANCELLED)` |
| Сегодня | `due_date = today` |
| Ближайшие 7 дней | `due_date BETWEEN today AND today+7` |
| Ближайшие 30 дней | `due_date BETWEEN today AND today+30` |
| Без срока | `due_date IS NULL` |
| Произвольный диапазон | `due_date BETWEEN :from AND :to` |
