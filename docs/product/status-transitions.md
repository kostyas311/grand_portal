# Матрица статусов и переходов

## Статусы карточки

| Код | Отображение | Описание |
|-----|-------------|----------|
| `NEW` | Новое | Карточка создана и ожидает начала работы |
| `IN_PROGRESS` | В работе | Исполнитель работает над задачей |
| `REVIEW` | На проверке | Результат загружен и ожидает проверки |
| `DONE` | Готово | Проверка завершена, карточка закрыта |
| `CANCELLED` | Отменено | Карточка отменена и больше не редактируется |

## Базовый граф переходов

```text
[NEW] ----------------------------> [CANCELLED]
  |
  v
[IN_PROGRESS] --------------------> [CANCELLED]
  |
  v (нужен результат и проверяющий)
[REVIEW]
  |
  +--> [DONE] (для карточки с протоколом проверки все пункты должны быть закрыты)
  |
  +--> [IN_PROGRESS] (возврат с замечаниями)
```

## Детальные правила переходов

### NEW -> IN_PROGRESS

- Кто может: участник рабочей карточки, руководитель, администратор
- Условия: дополнительных ограничений нет
- Что происходит: карточка считается взятой в работу

### IN_PROGRESS -> REVIEW

- Кто может: исполнитель карточки, руководитель, администратор
- Условия:
  - у карточки есть хотя бы один результат;
  - назначен проверяющий
- Что происходит: карточка переходит на этап проверки

### REVIEW -> DONE

- Кто может: проверяющий карточки, руководитель, администратор
- Условия:
  - у карточки есть актуальный результат
  - если к карточке прикреплён протокол проверки, все его пункты отмечены как пройденные
- Что происходит: карточка закрывается для редактирования

### REVIEW -> IN_PROGRESS

- Кто может: проверяющий карточки, руководитель, администратор
- Условия:
  - обязателен комментарий с замечаниями, если нет непройденных пунктов протокола
- Что происходит: карточка возвращается исполнителю на доработку

Если в карточке есть непройденные пункты протокола проверки, система автоматически дополняет комментарий списком этих пунктов.

### ANY -> CANCELLED

- Кто может: любая роль портала
- Условия:
  - должна быть указана причина отмены
- Что происходит: карточка закрывается и остаётся доступной только для просмотра и скачивания

## Блокировка редактирования

Карточка становится недоступной для изменений в статусах:

- `DONE`
- `CANCELLED`

Для закрытых карточек:

- нельзя менять поля карточки;
- нельзя добавлять и удалять материалы;
- нельзя добавлять новые результаты;
- можно просматривать историю и скачивать файлы.

## Дополнительные правила этапа REVIEW

- протокол проверки показывается в карточке только в статусе `REVIEW`;
- в статусе `REVIEW` снять протокол, инструкцию или компонент может только проверяющий карточки или администратор;
- после закрытия последнего пункта протокола портал может сразу предложить перевести карточку в `DONE`.

## Приоритеты

| Код | Отображение |
|-----|-------------|
| `OPTIONAL` | Желательно |
| `NORMAL` | В рабочем режиме |
| `URGENT` | Срочно |
| `CRITICAL` | Очень срочно |
