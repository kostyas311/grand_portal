# Product Brief — NormBase Portal

## Продукт

**Название:** NormBase Portal
**Тип:** Внутренний веб-портал для корпоративной локальной сети
**Версия:** MVP 1.0

## Проблема

Организация регулярно получает входящую документацию из разных источников.
Обработка этих документов ведётся командой, но:
- нет единого места хранения карточек задач
- нет трекинга статусов обработки
- нет версионирования результатов
- нет понятного рабочего процесса проверки
- файлы хранятся хаотично

## Решение

Внутренний портал с карточками задач, назначением ответственных, прикреплением файлов,
версионированием результатов и публичной страницей готовых проектов.

## Пользователи

| Пользователь | Потребность |
|---|---|
| Координатор | Создать карточку, назначить исполнителя, контролировать статус |
| Исполнитель | Взять карточку в работу, загрузить результат, отправить на проверку |
| Проверяющий | Скачать результат, одобрить или вернуть с замечаниями |
| Администратор | Управлять справочниками, пользователями, системой |

## Ценность

- Единое место для всех входящих документов
- Прозрачный статус обработки
- История версий результатов
- Публичная страница готовых проектов для доступа всей команды
- Аудит всех действий

## Ограничения MVP

- Только локальная сеть (без публичного доступа)
- Без интеграции с внешними системами (future: LDAP/AD)
- Без мобильного приложения (responsive web достаточно)
- Без email-уведомлений (future scope)
- Без real-time WebSocket нотификаций (future scope)

## Допущения

1. Система работает на одном сервере в локальной сети
2. Количество пользователей: до 50 одновременных
3. Количество карточек: до 10 000 в год
4. Размер файлов: до 500 MB на один файл, до 5 GB суммарно на карточку
5. Хранение файлов: локальная файловая система или сетевой диск (NFS/SMB)
6. Бэкапы организуются внешними средствами (cron + rsync или аналог)
7. Пользователи знают логин/пароль — самостоятельная регистрация не нужна (только Admin создаёт)
